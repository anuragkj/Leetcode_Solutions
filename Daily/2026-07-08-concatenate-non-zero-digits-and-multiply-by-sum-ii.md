## Original Problem

You are given a string s of length m consisting of digits. You are also given a 2D integer array queries, where queries[i] = [l_i, r_i].

For each queries[i], extract the substring s[l_i..r_i]. Then, perform the following:

- Form a new integer x by concatenating all the non-zero digits from the substring in their original order. If there are no non-zero digits, x = 0.

- Let sum be the sum of digits in x. The answer is x * sum.

Return an array of integers answer where answer[i] is the answer to the i^th query.

Since the answers may be very large, return them modulo 10^9 + 7.

Example 1:

Input: s = "10203004", queries = [[0,7],[1,3],[4,6]]

Output: [12340, 4, 9]

Explanation:

- s[0..7] = "10203004"

- x = 1234

- sum = 1 + 2 + 3 + 4 = 10

- Therefore, answer is 1234 * 10 = 12340.

- s[1..3] = "020"

- x = 2

- sum = 2

- Therefore, the answer is 2 * 2 = 4.

- s[4..6] = "300"

- x = 3

- sum = 3

- Therefore, the answer is 3 * 3 = 9.

Example 2:

Input: s = "1000", queries = [[0,3],[1,1]]

Output: [1, 0]

Explanation:

- s[0..3] = "1000"

- x = 1

- sum = 1

- Therefore, the answer is 1 * 1 = 1.

- s[1..1] = "0"

- x = 0

- sum = 0

- Therefore, the answer is 0 * 0 = 0.

Example 3:

Input: s = "9876543210", queries = [[0,9]]

Output: [444444137]

Explanation:

- s[0..9] = "9876543210"

- x = 987654321

- sum = 9 + 8 + 7 + 6 + 5 + 4 + 3 + 2 + 1 = 45

- Therefore, the answer is 987654321 * 45 = 44444444445.

- We return 44444444445 modulo (10^9 + 7) = 444444137.

Constraints:

- 1 <= m == s.length <= 10^5

- s consists of digits only.

- 1 <= queries.length <= 10^5

- queries[i] = [l_i, r_i]

- 0 <= l_i <= r_i < m

## Problem Summary
For each given query `[l, r]`, we extract the substring `s[l..r]`. From this substring, we form an integer `x` by concatenating all its non-zero digits in their original order (if no non-zero digits, `x=0`). We then calculate `sum`, the sum of digits in `x`. The final answer for the query is `x * sum`, returned modulo `10^9 + 7`.

## Intuition
The problem asks us to perform calculations on substrings, specifically involving filtering non-zero digits and then forming a number and its digit sum. A naive approach of iterating through the substring for each query would be too slow (`O(m)` per query, total `O(Q*m)`), given `m, Q <= 10^5`. This suggests we need a **precomputation** strategy to answer queries in `O(1)` or `O(log m)` time.

The key challenge is the "non-zero digits" requirement. Standard prefix sums on the original string `s` won't work directly because zeros are skipped. This leads to the central idea:
1.  **Focus only on non-zero digits**: Create a *compressed* representation of `s` that only contains its non-zero digits, along with their original indices.
2.  **Map queries to the compressed representation**: For any query `[l, r]` on `s`, we need to efficiently find the corresponding start and end indices (`start_k`, `end_k`) within our compressed list of non-zero digits.
3.  **Prefix sums on compressed data**: Once we have `start_k` and `end_k`, we can use prefix sums on the compressed list to quickly calculate the concatenated number `x` and its `sum` of digits for that range.

## Approach
The optimal solution involves several precomputation steps to answer each query in `O(1)` time.

1.  **Define Modulo**: Set `MOD = 10^9 + 7`. All calculations involving `x` and `sum` must be performed modulo `MOD`.

2.  **`nonzero_digits_info`**:
    *   Iterate through the input string `s`.
    *   For each non-zero digit `d` at index `i` in `s`, store `(d, i)` in a list called `nonzero_digits_info`. This list effectively "compresses" `s` by removing zeros.

3.  **`next_nz_in_compressed`**:
    *   Create an array `next_nz_in_compressed` of size `m` (length of `s`).
    *   `next_nz_in_compressed[i]` will store the **index** in `nonzero_digits_info` of the *first* non-zero digit in `s` that appears at or after original index `i`.
    *   If no such digit exists (e.g., `s[i..m-1]` contains only zeros), store `len(nonzero_digits_info)` (a sentinel value indicating "out of bounds").
    *   This array can be built by iterating `s` backwards, using a pointer to `nonzero_digits_info`.

4.  **`prev_nz_in_compressed`**:
    *   Create an array `prev_nz_in_compressed` of size `m`.
    *   `prev_nz_in_compressed[i]` will store the **index** in `nonzero_digits_info` of the *last* non-zero digit in `s` that appears at or before original index `i`.
    *   If no such digit exists (e.g., `s[0..i]` contains only zeros), store `-1` (a sentinel value).
    *   This array can be built by iterating `s` forwards, using a pointer to `nonzero_digits_info`.

5.  **Prefix Sums for `x` and `sum`**:
    *   `prefix_x`: An array where `prefix_x[k]` stores the integer formed by concatenating `nonzero_digits_info[0]` through `nonzero_digits_info[k-1]`, modulo `MOD`.
    *   `prefix_sum_digits`: An array where `prefix_sum_digits[k]` stores the sum of digits from `nonzero_digits_info[0]` through `nonzero_digits_info[k-1]`, modulo `MOD`.
    *   Both arrays will have size `len(nonzero_digits_info) + 1` to accommodate the "empty prefix" at index 0.

6.  **`prefix_pow10`**:
    *   An array `prefix_pow10` where `prefix_pow10[k]` stores `10^k % MOD`.
    *   This is used to efficiently calculate the value of a concatenated number from its prefix sums (e.g., `Value(A..B) = (Value(0..B) - Value(0..A-1) * 10^(length of A..B)) % MOD`).
    *   This array will have size `m + 1`, as the maximum length of `x` can be `m`.

7.  **Process Queries**: For each query `[l_i, r_i]`:
    *   Determine `start_k = next_nz_in_compressed[l_i]` and `end_k = prev_nz_in_compressed[r_i]`. These are the indices in `nonzero_digits_info` that correspond to the first and last non-zero digits within `s[l_i..r_i]`.
    *   **Handle Empty Range**: If `start_k > end_k`, it means there are no non-zero digits in `s[l_i..r_i]`. In this case, `x = 0`, `sum = 0`, and the answer is `0`.
    *   **Calculate `x`**: The number `x` for the range `nonzero_digits_info[start_k..end_k]` can be found using the prefix sums:
        `x = (prefix_x[end_k + 1] - (prefix_x[start_k] * prefix_pow10[end_k - start_k + 1]) % MOD + MOD) % MOD`
    *   **Calculate `sum`**: The sum of digits for the range `nonzero_digits_info[start_k..end_k]` is:
        `sum = (prefix_sum_digits[end_k + 1] - prefix_sum_digits[start_k] + MOD) % MOD`
    *   **Final Answer**: The result for the query is `(x * sum) % MOD`.
