## Original Problem

An integer has sequential digits if and only if each digit in the number is one more than the previous digit.

Return a sorted list of all the integers in the range [low, high] inclusive that have sequential digits.

Example 1:

Input: low = 100, high = 300
Output: [123,234]

Example 2:

Input: low = 1000, high = 13000
Output: [1234,2345,3456,4567,5678,6789,12345]

Constraints:

- 10 <= low <= high <= 10^9

**Examples / sample tests:**

```

```

## Problem Summary
This problem asks us to find all integers within a given inclusive range `[low, high]` that have "sequential digits". An integer has sequential digits if each digit in the number is one more than the previous digit (e.g., 123, 4567). The final list of these numbers must be sorted.

## Intuition
The key observation is that there are **very few** numbers with sequential digits. The smallest is 1, and the largest is 123456789. All these numbers are less than 10^9, which is the maximum value for `high`. This means we can **generate all possible sequential digit numbers** and then simply filter them based on the `[low, high]` range.

How do we generate them systematically?
We can start with any single digit from 1 to 9.
*   If we start with `1`, we can form `1`, `12`, `123`, ..., `123456789`.
*   If we start with `2`, we can form `2`, `23`, `234`, ..., `23456789`.
*   And so on, up to `9`, which only forms `9`.

This generation process resembles a **Breadth-First Search (BFS)**. We start with single digits, then extend them to two-digit numbers, then three-digit numbers, and so on. BFS naturally explores numbers by their length first, and within the same length, by their value, which means the numbers will be generated in **sorted order**.

## Approach
We will use a **Breadth-First Search (BFS)** approach to generate all sequential digit numbers in increasing order and filter them within the given range.

1.  **Initialize**:
    *   Create an empty list `result` to store the sequential digit numbers that fall within `[low, high]`.
    *   Create a `collections.deque` (a double-ended queue) and initialize it with single digits `1` through `9`. These are our starting points for generating sequential numbers.

2.  **BFS Traversal**:
    *   While the `queue` is not empty:
        a.  **Dequeue**: Remove the `current_num` from the front of the `queue`.
        b.  **Check Range**: If `current_num` is within the `[low, high]` range (i.e., `low <= current_num <= high`), add it to our `result` list.
        c.  **Pruning (High-end)**: If `current_num` is already greater than `high`, then any longer sequential numbers built from `current_num` will also be greater than `high`. So, we can stop exploring this branch and `continue` to the next number in the queue.
        d.  **Generate Next**: Get the `last_digit` of `current_num` (e.g., for 123, `last_digit` is 3).
        e.  **Extend Number**: If `last_digit` is less than `9` (meaning we can still append a larger digit), calculate `next_num = current_num * 10 + (last_digit + 1)`.
        f.  **Pruning (Next Number)**: If `next_num` is still less than or equal to `high` (or even better, less than or equal to the maximum possible sequential digit number, 123456789), add `next_num` to the back of the `queue` to be processed later. This prevents adding numbers that are too large too early.

3.  **Return Result**: After the BFS completes, the `result` list will contain all sequential digit numbers within `[low, high]`, and they will already be sorted due to the nature of BFS. Return `result`.

## Problem Summary
We need to find all integers within a given range `[low, high]` (inclusive) that have "sequential digits". An integer has sequential digits if each digit is exactly one greater than the preceding digit (e.g., 123, 4567). The final list of these numbers should be sorted in ascending order.

## Intuition
The key insight is that there are very few numbers with sequential digits.
*   The smallest is `1`.
*   The largest is `123456789`.
*   All sequential digit numbers are less than 10^9 (the maximum `high` value).

This means we can **generate all possible sequential digit numbers** and then simply filter them to see which ones fall within the `[low, high]` range.

How do we generate them systematically? We can think of building these numbers digit by digit:
1.  Start with single digits: `1, 2, 3, ..., 9`.
2.  From `1`, we can form `12` (by appending `2`).
3.  From `12`, we can form `123` (by appending `3`).
4.  ...and so on, until we can no longer append a digit (i.e., the last digit is `9`).

This process of exploring numbers by extending them resembles a **Breadth-First Search (BFS)**. If we start our BFS with `1, 2, ..., 9` in the queue, it will naturally generate numbers in increasing order (first by length, then by value). This is a huge advantage as it means our final `result` list will already be sorted without needing an explicit sort step.

## Approach
We will use a **Breadth-First Search (BFS)** algorithm to generate sequential digit numbers. This approach ensures that numbers are generated in ascending order, making the final result naturally sorted.

1.  **Initialization**:
    *   Create an empty list called `result` to store all sequential digit numbers that fall within the `[low, high]` range.
    *   Initialize a `collections.deque` (a double-ended queue) named `queue`. Populate it with the single digits from `1` to `9`. These single digits serve as the starting points for generating longer sequential numbers.

2.  **BFS Traversal**:
    *   Begin a loop that continues as long as the `queue` is not empty.
    *   Inside the loop:
        a.  **Dequeue Current Number**: Remove the `current_num` from the front of the `queue` using `queue.popleft()`.
        b.  **Range Check and Add**: Check if `current_num` is within the specified range: `low <= current_num <= high`. If it is, add `current_num` to our `result` list.
        c.  **High-End Pruning**: If `current_num` is already greater than `high`, it means any further sequential numbers built by extending `current_num` will also be greater than `high`. In this case, there's no need to explore this branch further, so we `continue` to the next number in the queue. This is an important optimization to avoid unnecessary computations.
        d.  **Determine Next Digit**: Get the `last_digit` of `current_num` by using the modulo operator: `last_digit = current_num % 10`.
        e.  **Extend Number**: If `last_digit` is less than `9` (meaning we can still append a digit that is one greater), calculate the `next_num` by appending `last_digit + 1`: `next_num = current_num * 10 + (last_digit + 1)`.
        f.  **Enqueue Next Number (with Pruning)**: Before adding `next_num` to the queue, check if it's still potentially relevant. If `next_num` is less than or equal to `high`, then enqueue `next_num` to be processed in a future iteration. This prevents adding numbers to the queue that are already too large for our target range.

3.  **Return Result**: Once the `queue` becomes empty, the BFS traversal is complete. The `result` list will contain all sequential digit numbers within `[low, high]`, and they will be in ascending order. Return the `result` list.

## Solution

```python
import collections
from typing import List

class Solution:
    def sequentialDigits(self, low: int, high: int) -> List[int]:
        result = []
        # Initialize a queue for BFS with single digits 1 through 9.
        # These are the starting points for generating sequential numbers.
        queue = collections.deque(range(1, 10))

        while queue:
            current_num = queue.popleft()

            # If the current number is within the [low, high] range, add it to our results.
            if low <= current_num <= high:
                result.append(current_num)
            
            # Optimization: If current_num already exceeds high,
            # any further numbers built from it will also exceed high.
            # So, we can stop exploring this branch.
            if current_num > high:
                continue # Skip to the next number in the queue

            # Get the last digit of the current number.
            last_digit = current_num % 10

            # If the last digit is less than 9, we can append the next sequential digit.
            if last_digit < 9:
                next_num = current_num * 10 + (last_digit + 1)
                
                # Only enqueue if the next number is still potentially in range.
                # This further prunes the search space.
                if next_num <= high:
                    queue.append(next_num)
        
        # The BFS approach naturally generates numbers in increasing order (by length, then by value),
        # so the 'result' list will already be sorted.
        return result

```

## Why This Works
This solution works because it systematically generates **all possible sequential digit numbers** using a Breadth-First Search (BFS) strategy. BFS explores numbers level by level (first all 1-digit, then all 2-digit, etc.), and within each level, it processes numbers in ascending order. This guarantees that any numbers added to our `result` list will automatically be in sorted order. The crucial **pruning steps** (`if current_num > high: continue` and `if next_num <= high: queue.append(next_num)`) efficiently cut off branches of the search tree that would only produce numbers outside the `[low, high]` range, ensuring optimal performance. Since the total count of sequential digit numbers is very small and fixed (only 45), this approach is extremely efficient, effectively running in constant time and space.
